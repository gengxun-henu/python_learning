## 先建立正确心态：你不是从零开始

你不需要从 `if`、`for`、类和函数这些基础概念重新学起。你需要重点适应的是：

- **语法风格变化**：缩进即结构，代码更短更直白
- **类型与对象模型**：动态类型、名字绑定、可变/不可变对象
- **标准库与生态**：很多事 Python 已经帮你封装好了
- **工程方式变化**：虚拟环境、包管理、脚本化、交互式开发
- **性能思维变化**：少写“手搓底层循环”，多用内建、库、向量化

如果用类比来说：

- C++ 更像“精密机械加工”
- Python 更像“搭积木 + 自动化流水线”

两者都强，只是发力点不同。

## 最重要的 8 个学习重点

## 1. 先学“Pythonic”写法，不要直接把 C++ 翻译过来

初学 Python 最常见的问题，就是写出“带 Python 语法的 C++”。

比如你在 C++ 里习惯：

- 明确类型
- 手动控制对象生命周期
- 写很多模板化、结构化、层次化代码
- 喜欢显式、严谨、可控

Python 里要适应：

- **能直接表达意图，就不要绕**
- **优先可读性**
- **优先用语言和库现成能力**
- **少写样板代码**

Python 的设计哲学里有一句很有代表性的话：

> There should be one—and preferably only one—obvious way to do it.

所以一开始别急着追求“面向对象架构感”，先学会：

- 列表推导式
- 字典/集合操作
- `for x in iterable`
- `with`
- `enumerate`
- `zip`
- `pathlib`
- `collections`
- `dataclasses`

这些东西会让你很快体会到 Python 的味道。

---

## 2. 理解变量不是“盒子”，而是“名字绑定”

这是 C++ 程序员转 Python 最需要真正吃透的一点。

在 Python 里：

- 变量更像**名字**
- 对象独立存在
- 赋值是**绑定**
- 不是“拷贝对象内容”这个直觉

比如：

- `a = [1, 2]`
- `b = a`

这通常不是“复制了一个 list”，而是 `a` 和 `b` 指向同一个对象。

所以必须重点理解：

- **可变对象**：`list`、`dict`、`set`
- **不可变对象**：`int`、`float`、`str`、`tuple`
- **身份 vs 值**
  - `is`：是不是同一个对象
  - `==`：值是否相等

这和 C++ 的值语义/引用语义思维不完全一样。  
你可以把 Python 看成：**默认到处都在传对象引用**，但语法上把这件事藏得很轻。

---

## 3. 类型系统：动态类型不等于不要类型 discipline

你有 C++ 背景，可能会担心：“动态类型会不会太散？”

答案是：**运行时是动态类型，但工程上完全可以写得很规范。**

现代 Python 的推荐做法是：

- 日常代码继续享受动态语言的灵活性
- 对核心接口、公共函数、复杂数据结构加上 **type hints**
- 配合静态检查工具：
  - `pyright`
  - `mypy`

你可以这样理解：

- C++：类型系统是编译器强制执行的主逻辑
- Python：类型提示是**增强可读性、工具支持和工程质量**的辅助系统

你不必一上来把所有代码写成“伪静态语言”，但建议尽早建立这个习惯：

- 函数参数写类型
- 返回值写类型
- 复杂结构用 `dataclass`
- 协议接口可看 `Protocol`

这会非常适合你这种有大型工程经验的人。

---

## 4. 容器和迭代模型要熟

C++ 老手通常很快能上手 Python 容器，但要注意“常用姿势”不同。

### 常用容器对照

- `list` ≈ `std::vector`
- `tuple` ≈ 不可变轻量记录 / 固定长度组合
- `dict` ≈ `std::unordered_map`
- `set` ≈ `std::unordered_set`

### 重点不是容器本身，而是“迭代方式”

Python 的核心风格是：

- 面向 iterable / iterator 编程
- 少用下标，多直接遍历
- 少写索引控制，多用工具函数

优先掌握这些：

- `for x in items`
- `for i, x in enumerate(items)`
- `for a, b in zip(xs, ys)`
- `if x in s`
- 列表/字典/集合推导式
- 生成器表达式
- `sorted(..., key=...)`

这部分学熟了，写 Python 会顺很多。

---

## 5. 函数是一等公民，默认参数和闭包要注意坑

Python 里函数比 C++ 里更“灵活”：

- 可以作为参数传递
- 可以作为返回值返回
- 可以嵌套定义
- 可以形成闭包

但有几个经典坑需要特别记住。

### 坑 1：可变默认参数

不要写“默认空列表/空字典”然后在里面改。  
因为默认参数是在函数定义时创建的，不是每次调用都新建。

### 坑 2：闭包晚绑定

循环里创建 lambda/内部函数时，变量绑定时机常和直觉不同。

### 坑 3：参数传递方式要搞清楚

要熟悉：

- 位置参数
- 关键字参数
- 默认参数
- `*args`
- `**kwargs`

这部分建议你早点学，因为 Python 代码风格里函数式技巧很常见。

---

## 6. 面向对象要学，但不要用 C++ 的重量级方式去套

Python 支持 OOP，但整体是“轻量派”。

### 要掌握的类概念

- `class`
- `self`
- 实例属性 / 类属性
- 继承
- `super()`
- 魔术方法：
  - `__init__`
  - `__repr__`
  - `__len__`
  - `__iter__`
  - `__call__`
  - `__enter__` / `__exit__`

### 特别推荐先学

- `dataclasses.dataclass`
- `property`
- `abc`
- `Protocol`（如果你对接口设计有要求）

### 思维差异

C++ 里你可能会很自然地：

- 设计复杂类层次
- 抽象多个模板和继承层
- 强调对象边界和封装粒度

Python 里通常更推荐：

- 简洁对象
- 扁平结构
- 组合优先
- 少写“为了架构而架构”的层次

一句话：**Python 也能做大工程，但它的大工程不是靠“重类层次”来显得专业。**

---

## 7. 异常处理和上下文管理非常重要

你有 C++ 背景，所以这里非常适合用 RAII 去类比。

Python 里：

- 错误主要靠 `try/except`
- 资源管理的核心工具是 `with`

你可以把 `with` 理解成一种更高层、更统一、更常见的 RAII 风格接口。

常见场景：

- 文件打开关闭
- 锁管理
- 数据库连接
- 临时资源
- 事务控制

这是 Python 非常“优雅”的一部分，务必学扎实。

## 8. 工程化不是补充，而是主线

初学 Python 的人容易只学语法，不学环境和项目组织，后面就会比较乱。

你应该尽早掌握这些工程要点：

### 环境管理

- `venv`：标准虚拟环境
- 包安装：`pip`
- 项目配置：`pyproject.toml`

### 常见工具链

- 格式化：`black`
- lint：`ruff`
- 测试：`pytest`
- 类型检查：`pyright` 或 `mypy`

### 推荐目录意识

- 模块和包怎么组织
- `__init__.py` 的作用
- 相对导入 / 绝对导入
- 脚本入口：`if __name__ == "__main__":`

如果你想快速进入“写得像个成熟 Python 项目”的状态，这一块非常值得早点学。

## C++ 程序员最容易踩的坑

这部分我单独拎出来，因为很实用。

### 1. 不要把每个东西都设计成类

很多 Python 问题用：

- 函数
- 模块
- 简单数据结构

就够了。

### 2. 不要过度手写循环

很多情况下，Python 里更推荐：

- 推导式
- 内建函数
- 标准库
- 第三方库

而不是手工写很多层循环。

### 3. 不要默认认为赋值就是复制

这点前面说过，是 Python 新手常见 bug 来源。

### 4. 不要忽视可变对象副作用

尤其是：

- list
- dict
- set

函数传参、成员属性、默认参数里都要留神。

### 5. 不要为了“像 C++ 一样严谨”把代码写得过重

Python 的优势在于：

- 快速表达
- 清晰可读
- 开发效率高

不是靠写很多 ceremony 来体现专业。

### 6. 不要一开始就纠结性能

你有 C++ 背景，天然会敏感性能，这很好；但 Python 里更重要的是：

1. 先写对
2. 再 profile
3. 再优化热点
4. 优先借助高性能库

很多时候真正的优化路线不是“把 Python 写得更像 C++”，而是：

- 用内建函数
- 用 `numpy`
- 用 `pandas`
- 用 C 扩展/绑定
- 用多进程
- 把热点下沉到 C/C++/Rust
