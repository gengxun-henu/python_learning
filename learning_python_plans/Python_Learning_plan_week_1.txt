一周学习计划：给有 20 年 C++ 经验者
下面这套按 每天 1~2 小时 设计，强调：

快速建立手感
避免陷入“只看不写”
每天都产出一点可运行代码
第 1 天：建立语感，先能读能写
目标
看懂基本 Python 代码
会写最简单脚本
学习内容
变量与基本类型：int, float, bool, str
if/elif/else
for / while
缩进规则
输入输出：print()
重点理解
Python 没有你熟悉的 {}，结构全靠缩进
变量不声明类型
代码往往比 C++ 短很多，这是正常现象，不是“偷工减料”
练习
写一个小程序：

输入一个整数 n
输出 
1
1 到 
n
n 的和
判断奇偶
输出平方表
今天必须记住
冒号 :
缩进
for x in range(...)
第 2 天：掌握 4 大核心容器
目标
熟练用 list / tuple / dict / set
学习内容
创建、访问、修改
遍历
成员判断
基本方法
重点理解
list：最常用
dict：极高频
set：查重神器
tuple：不可变、轻量
练习
做 3 个小题：

统计字符串中每个字符出现次数
对列表去重
按学生姓名映射成绩，并找最高分
今天必须记住
dict 是 Python 主战容器
in 很常用
少用“纯下标控制式思维”
第 3 天：函数、模块、文件读写
目标
能把代码拆成函数
能读写文件
理解模块导入
学习内容
def
参数、返回值
默认参数
import
open() 和 with
重点理解
函数是组织 Python 代码的第一工具
with open(...) as f: 是标准姿势
默认参数不要乱放可变对象
练习
写一个文本文件统计脚本：

读取文本
统计行数、单词数、字符数
输出前 10 个高频词
今天必须记住
with
return
模块导入
可变默认参数是坑
第 4 天：Pythonic 写法入门
目标
开始摆脱 C++ 风格 Python
学习内容
列表推导式
字典推导式
enumerate
zip
sorted(key=...)
切片
字符串格式化 f-string
重点理解
这是“像 Python”最重要的一天。
你会第一次明显感觉：原来很多代码不用写那么长。

练习
把前几天的练习代码重写一遍：

能用推导式的地方改成推导式
能直接遍历的地方不再手写下标循环
输出全部改成 f-string
今天必须记住
Pythonic 不是“炫技”，而是“更清晰”
enumerate 和 zip 非常值钱
第 5 天：对象模型、类、dataclass
目标
学会 Python 中“够用而优雅”的 OOP
学习内容
class
self
__init__
__repr__
@dataclass
property（了解即可）
重点理解
Python 支持 OOP，但别搞得像大型 C++ 模板框架开幕式
很多简单数据对象直接 @dataclass 就够了
练习
实现一个小模型：

Point
Rectangle
支持面积计算
打印友好字符串
判断点是否在矩形内
今天必须记住
self
@dataclass
组合通常优于复杂继承
第 6 天：异常、资源管理、工程化起步
目标
学会写“像项目”的代码
学习内容
try/except/finally
raise
with
虚拟环境概念
pytest
格式化/lint/类型提示的角色
重点理解
Python 项目质量，不靠“写很多模板代码”
而靠：测试、格式、检查、清晰结构
练习
给前面的文本统计脚本加上：

文件不存在异常处理
空文件处理
一个简单测试函数/测试文件思路
今天必须记住
with 是资源管理核心
pytest 值得尽早上手
类型提示是工程增强器，不是负担

对你最有价值的学习策略
以你的背景，最有效的不是“看很多基础视频”，而是：

1. 用“对照学习”
见到新东西就问自己：

这在 C++ 里对应什么？
Python 为什么这样设计？
什么时候应该用 Python 式做法，而不是 C++ 式做法？
2. 小步快跑
每天写 50~100 行，比看 3 小时教程更有效。

3. 尽早接触真实项目结构
哪怕是小脚本，也要试着：

分函数
分模块
加一点测试
加类型注解
4. 不要和 Python 的动态性对抗
不是让你放弃工程纪律，而是要学会：

在灵活和规范之间拿平衡

名字绑定 vs 复制
可变对象副作用
list/dict/set 熟练使用
直接遍历、enumerate、zip
推导式
with
函数拆分
dataclass
异常处理
pytest + type hints + venv 的工程化意识
