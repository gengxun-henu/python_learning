# C++ → Python 迁移速查表

> 适用对象：有长期 C++ 经验，准备快速掌握 Python 编程核心思维的开发者。

## 先记住这 5 条总原则

| C++ 直觉 | Python 直觉 |
|---|---|
| 类型先行 | 意图先行 |
| 值语义和对象生命周期很重要 | 名字绑定和对象可变性很重要 |
| 设计结构后实现 | 先写清楚，再逐步抽象 |
| 手工控制性能 | 先借助内建和库，再优化热点 |
| 模板/泛型常是核心工具 | Duck typing + type hints + 协议更常见 |

---

## 1. 基本语法与代码结构

| C++ | Python | 迁移提示 |
|---|---|---|
| `{}` 表示作用域 | 缩进表示作用域 | Python 对缩进非常敏感 |
| `;` 结尾 | 不需要 `;` | 不要下意识补分号 |
| `#include` | `import` | 模块导入比头文件轻得多 |
| `main()` | `if __name__ == "__main__":` | 脚本入口常见写法 |
| `//`、`/* */` | `#`、三引号字符串 | 文档字符串很常见 |

---

## 2. 变量、对象、赋值

| C++ | Python | 迁移提示 |
|---|---|---|
| `int x = 3;` | `x = 3` | 变量不显式声明类型 |
| 值拷贝常见 | 赋值通常是绑定名字 | `b = a` 常常不是复制 |
| 引用/指针显式 | 对象引用更隐式 | 很容易产生共享引用问题 |
| `const` 很重要 | 不可变对象更重要 | `tuple`、`str`、`int` 都是不可变对象 |

### 必记

- `==`：值相等
- `is`：是否是同一个对象

这两个在 Python 中一定要分清楚。

---

## 3. 常见数据结构对照

| C++ | Python | 说明 |
|---|---|---|
| `std::vector<T>` | `list` | 最常用的动态数组 |
| `std::array<T, N>` | `tuple` / `list` | `tuple` 更偏固定、不可变 |
| `std::unordered_map<K, V>` | `dict` | Python 核心容器之一 |
| `std::unordered_set<T>` | `set` | 去重和成员判断非常方便 |
| `std::map<K, V>` | `dict` | 现代 `dict` 保留插入顺序 |
| `std::pair<A, B>` | `tuple` | 解包非常方便 |

### Python 常见遍历方式

- `for x in items`
- `for i, x in enumerate(items)`
- `for a, b in zip(xs, ys)`

> Python 更强调“直接遍历对象”，而不是手工控制下标。

---

## 4. 条件与循环

| C++ | Python | 迁移提示 |
|---|---|---|
| `if (...) {}` | `if ...:` | 冒号 + 缩进 |
| `for (auto& x : v)` | `for x in v:` | 与范围 for 很接近 |
| 经典下标循环常见 | 下标循环相对少 | 优先直接遍历 |
| `switch` | `match`（3.10+）/ `if-elif` | 多数场景 `if-elif` 就够了 |
| `break / continue` | `break / continue` | 基本一致 |

---

## 5. 字符串处理

| C++ | Python | 迁移提示 |
|---|---|---|
| `std::string` | `str` | Python 字符串能力非常强 |
| `+` 拼接 | `+` / `''.join(...)` | 多次拼接优先 `join` |
| `std::format` / iostream | `f-string` | 最推荐的格式化方式 |
| 手写解析 | `split`、`strip`、`replace` | 标准库已覆盖大量场景 |

### 最推荐先熟练的接口

- `f"{name} = {value}"`
- `split()`
- `strip()`
- `startswith()`
- `endswith()`

---

## 6. 函数与参数

| C++ | Python | 迁移提示 |
|---|---|---|
| `int add(int a, int b)` | `def add(a: int, b: int) -> int:` | 推荐加类型注解 |
| 函数重载常见 | 没有传统意义上的重载 | 常用默认参数和可变参数 |
| 模板函数 | Duck typing / type hints / `TypeVar` | 工程层面增强 |
| 默认参数 | 默认参数 | 可变默认值是经典坑 |
| 可变参数模板 | `*args`, `**kwargs` | 很常见 |

### 大坑提醒

不要把空列表、空字典直接作为默认参数并在函数里修改。

---

## 7. 类与面向对象

| C++ | Python | 迁移提示 |
|---|---|---|
| `class` | `class` | 语法更轻 |
| `this` | `self` | 需要显式写出来 |
| 构造函数 | `__init__` | 更接近初始化方法 |
| 析构函数 | `__del__` | 不要用来依赖关键资源释放 |
| `public/private/protected` | `_name` / `__name` 约定式控制 | Python 更信任开发者 |
| 运算符重载 | 魔术方法 | 如 `__repr__`、`__len__`、`__iter__` |

### 最值得优先学

- `@dataclass`
- `property`
- `super()`

### 迁移建议

Python 支持 OOP，但通常更强调：

- 轻量类
- 组合优先
- 模块级函数同样重要
- 用 `dataclass` 减少样板代码

---

## 8. 异常处理

| C++ | Python | 迁移提示 |
|---|---|---|
| `try/catch` | `try/except` | 十分相似 |
| `throw` | `raise` | 语义直观 |
| 异常类型体系 | 异常类体系 | 推荐捕获具体异常 |
| 错误码风格 | Python 更偏异常机制 | “失败即抛异常”更常见 |

### 推荐习惯

- 不要滥用裸 `except:`
- 优先捕获具体异常类型
- 使用 `finally` 做清理
- 使用 `raise ... from e` 保留异常链

---

## 9. 资源管理：RAII 在 Python 中的对应物

| C++ | Python | 迁移提示 |
|---|---|---|
| RAII | `with` + 上下文管理器 | 最核心的对应关系 |
| 构造获取资源、析构释放 | `__enter__` / `__exit__` | Pythonic 资源管理接口 |
| `std::lock_guard` | `with lock:` | 非常像 |
| 文件自动关闭 | `with open(...) as f:` | 标准用法 |

### 核心认知

- 不要依赖 `__del__` 做关键资源释放
- 优先用 `with` 管理文件、锁、连接等资源

> 可以直接把 `with` 理解成 Python 版通用 RAII 入口。

---

## 10. 泛型 / 模板思维迁移

| C++ | Python | 迁移提示 |
|---|---|---|
| 模板是核心抽象工具 | Duck typing 很常见 | “像鸭子就行” |
| `template<typename T>` | `TypeVar`、`Generic` | 更多用于工程增强 |
| Concepts | `Protocol` | 很适合表达结构化接口 |
| 模板特化 | Python 很少这么做 | 更常见的是分发、注册、鸭子类型 |

### 迁移建议

在 Python 中常见做法是：

1. 先写通用实现
2. 用文档和类型提示说明接口
3. 必要时用 `Protocol` 约束“需要有哪些能力”

### 核心思维转变

- C++：更关注“它是什么类型”
- Python：更关注“它能做什么”

---

## 11. 模块、包、命名空间

| C++ | Python | 迁移提示 |
|---|---|---|
| `namespace` | 模块 / 包 | Python 更依赖文件系统组织 |
| `.h/.cpp` 分离 | 一个 `.py` 文件就是一个模块 | 结构更轻 |
| 编译链接 | 解释执行 + 导入 | 迭代速度快很多 |

### 要熟悉的导入方式

- `import a`
- `from a import b`
- 包目录中的 `__init__.py`

---

## 12. 并发模型

| C++ | Python | 迁移提示 |
|---|---|---|
| `std::thread` | `threading` | I/O 场景常用 |
| 多进程 | `multiprocessing` | CPU 密集场景更常见 |
| 异步生态相对弱一些 | `asyncio` 很重要 | 网络和高并发场景常见 |
| 真多线程并行常见 | 受 GIL 影响 | CPU 密集别迷信线程 |

### 快速记忆

- I/O 密集：线程或 `asyncio`
- CPU 密集：多进程、NumPy、C/C++ 扩展

---

## 13. 性能优化思路

| C++ 习惯 | Python 推荐 |
|---|---|
| 提前精打细算 | 先写清楚 |
| 微优化代码结构 | 先 profile |
| 手写高性能循环 | 优先内建函数和高性能库 |
| 靠语言本体榨性能 | 借助 `numpy`、`pandas`、C/C++ 扩展 |

### 对你最合适的路线

如果后续要结合你的 C++ 优势，最推荐的路线往往是：

> Python 负责组织业务流程，C++ 负责性能热点。

---

## 14. 工程工具链

| 目标 | Python 常用工具 |
|---|---|
| 虚拟环境 | `venv` |
| 包管理 | `pip` |
| 测试 | `pytest` |
| 格式化 | `black` |
| lint | `ruff` |
| 类型检查 | `pyright` / `mypy` |

### 推荐理解

Python 工程质量通常来自：

- 清晰的项目结构
- 自动化测试
- 格式化和静态检查
- 适量类型注解

而不是靠大量样板代码来体现“专业感”。

---

## 15. C++ 老手最容易踩的坑

1. 过度设计类层次
2. 把 Python 写成“没有类型声明的 C++”
3. 忽视可变对象副作用
4. 过早焦虑性能
5. 不愿使用标准库和第三方生态
6. 总想显式控制所有细节

Python 的核心优势之一，就是把很多重复劳动从你手里接过去。

---

## 一页版记忆口诀

- 变量是绑定，不是盒子
- 迭代优先，少写下标循环
- `dict / list / set` 是主力容器
- `with` 是 Python 的资源管理核心
- `dataclass` 很香
- Duck typing + type hints 比生搬模板思维更实用
- 先 profile，再优化
- 多用标准库，少手搓样板代码

---

## 最后一句建议

如果你想学得快，不要只问“Python 语法怎么写”，更要问：

> 这个问题在 Python 里，最自然、最简洁、最可维护的写法是什么？

这就是从 C++ 迁移到 Python 时最关键的学习转变。  
欢迎来到缩进和表达力并存的世界。
