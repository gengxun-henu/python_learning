# Python 学习计划：第 1 周

> 适用对象：有 20 多年 C++ 经验、刚开始系统学习 Python 的开发者  
> 建议节奏：每天 1~2 小时  
> 学习目标：快速建立 Python 语感，避免写出“C++ 风格的 Python”，并具备完成小型脚本和基础工程化组织的能力。

## 本周总目标

这一周不追求“学完 Python”，而是建立以下核心直觉：

- 理解 Python 的语法风格和缩进规则
- 熟悉 `list`、`dict`、`set`、`tuple` 四大核心容器
- 掌握函数、模块、文件读写、异常处理
- 初步形成 Pythonic 写法意识
- 了解 `class`、`dataclass`、`with`、`pytest` 的基本用途
- 能写出一个结构清楚的小工具程序

---

## 第 1 天：建立语感，先能读能写

### 今日目标

- 看懂最基本的 Python 代码
- 会写简单脚本
- 适应缩进语法和无类型声明的风格

### 学习内容

- 基本类型：`int`、`float`、`bool`、`str`
- 变量赋值
- `if / elif / else`
- `for`、`while`
- `print()`
- `range()`

### 今日重点

- Python 没有 `{}`，代码结构依赖缩进
- 变量不需要像 C++ 那样预先声明类型
- 同样的逻辑，Python 代码通常会比 C++ 短很多

### 建议练习

写一个小程序，完成以下功能：

1. 输入一个整数 `n`
2. 输出 $1$ 到 $n$ 的和
3. 判断 `n` 是奇数还是偶数
4. 输出 `1` 到 `n` 的平方表

### 今日必须记住

- 冒号 `:`
- 缩进必须统一
- `for x in range(...)` 是最常用循环方式之一

---

## 第 2 天：掌握 4 大核心容器

### 今日目标

- 熟练理解并使用 `list`、`tuple`、`dict`、`set`
- 体会 Python 数据结构的高表达力

### 学习内容

- 创建、访问、修改容器
- 遍历容器
- 成员判断：`in`
- 常用方法：`append`、`get`、`items`、`add`

### 今日重点

- `list`：最常用的动态数组
- `dict`：最核心的映射类型
- `set`：用于去重和快速成员判断
- `tuple`：不可变、轻量，适合固定结构数据

### 建议练习

完成以下 3 个小题：

1. 统计字符串中每个字符出现的次数
2. 对一个整数列表去重并保留唯一值
3. 用字典保存学生成绩，并找出最高分学生

### 今日必须记住

- `dict` 是 Python 里极高频容器
- `in` 非常常用
- 不要总想着用下标控制遍历

---

## 第 3 天：函数、模块、文件读写

### 今日目标

- 能把代码拆分成函数
- 会从文件读取数据并输出结果
- 理解 Python 模块导入方式

### 学习内容

- `def`
- 参数与返回值
- 默认参数
- `import`
- `with open(...) as f:`

### 今日重点

- 函数是组织 Python 代码的第一工具
- 文件读写优先使用 `with`
- 默认参数不要乱用可变对象（比如空列表、空字典）

### 建议练习

写一个文本统计脚本：

- 读取一个文本文件
- 统计行数、单词数、字符数
- 输出前 10 个高频词

### 今日必须记住

- `with`
- `return`
- `import`
- 可变默认参数是经典坑点

---

## 第 4 天：Pythonic 写法入门

### 今日目标

- 开始摆脱“C++ 风格 Python”
- 初步形成更简洁的表达方式

### 学习内容

- 列表推导式
- 字典推导式
- `enumerate`
- `zip`
- `sorted(..., key=...)`
- 切片
- `f-string`

### 今日重点

这一天非常关键，因为你会第一次明显感受到：

> 原来 Python 不是“把 C++ 简写一下”，而是一种完全不同的表达方式。

### 建议练习

把前几天的练习代码重写一遍：

- 能用推导式的地方改成推导式
- 能直接遍历的地方不用手写下标循环
- 所有输出尽量改成 `f-string`

### 今日必须记住

- Pythonic 不等于炫技，而是更清晰
- `enumerate` 和 `zip` 非常值得熟练掌握

---

## 第 5 天：对象模型、类、`dataclass`

### 今日目标

- 学会 Python 中够用、简洁的面向对象写法
- 理解 `self`、初始化方法和数据类

### 学习内容

- `class`
- `self`
- `__init__`
- `__repr__`
- `@dataclass`
- `property`（先了解）

### 今日重点

- Python 支持 OOP，但通常更轻量
- 简单数据对象优先考虑 `@dataclass`
- 不必一上来就设计复杂继承体系

### 建议练习

实现一个简单几何模型：

- `Point`
- `Rectangle`
- 支持面积计算
- 打印友好字符串
- 判断点是否落在矩形内部

### 今日必须记住

- `self`
- `@dataclass`
- 组合通常优于复杂继承

---

## 第 6 天：异常、资源管理、工程化起步

### 今日目标

- 学会写更稳健的 Python 代码
- 对 Python 的工程化方式建立基本认识

### 学习内容

- `try / except / finally`
- `raise`
- `with`
- 虚拟环境概念：`venv`
- 测试工具：`pytest`
- 格式化、lint、类型提示的角色

### 今日重点

- Python 中错误处理以异常机制为主
- `with` 是资源管理核心
- 工程质量来自测试、结构、格式化和检查，而不是样板代码堆砌

### 建议练习

为前面的文本统计脚本补充：

- 文件不存在时的异常处理
- 空文件处理
- 一个简单测试用例设计思路

### 今日必须记住

- `with` 是 Python 的资源管理利器
- `pytest` 越早接触越好
- 类型提示是工程增强器，不是负担

## 第 7 天：完成一个小项目

### 今日目标

- 把前 6 天的知识串起来
- 做出一个结构清晰、可运行的小工具

### 项目建议

做一个“目录文件统计工具”，支持以下功能：

- 遍历目录
- 统计文件数量
- 按扩展名分类
- 输出最大文件
- 支持简单命令行参数

### 涉及知识点

- `pathlib`
- 函数拆分
- `dict`
- 异常处理
- `with`
- 格式化输出

### 今日重点

今天最重要的不是“做得多复杂”，而是：

- 代码结构清楚
- 命名清楚
- 输出清楚
- 两天后再看仍然看得懂

## 本周最值得优先建立的 10 个直觉

1. 变量是“名字绑定”，不是“盒子”
2. 可变对象有副作用，要警惕共享引用
3. `list / dict / set` 是主力容器
4. 优先直接遍历，不要总写下标循环
5. 推导式很重要
6. `with` 是资源管理核心工具
7. 函数拆分优于一大段脚本堆叠
8. `dataclass` 很适合轻量建模
9. 异常处理是常规手段，不是补救措施
10. `pytest + type hints + venv` 是工程化起点

---

## 每天学习时的建议节奏

你可以把每天 1~2 小时拆成这样：

- 20 分钟：看语法/概念
- 20 分钟：看别人示例
- 30~60 分钟：自己写代码练习
- 10 分钟：总结今天学到的坑点和对照关系

这样会比只看教程高效很多。

---

## 结束语

这一周的核心不是“掌握所有 Python 细节”，而是完成从 C++ 思维到 Python 思维的第一步迁移。

记住一句最实用的话：

> 在 Python 里，先写清楚，再写复杂；先用标准库，再谈造轮子。
