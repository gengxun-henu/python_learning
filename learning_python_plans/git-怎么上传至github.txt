## 快速上手（本地仓库）

在你的项目目录里执行：

```bash
cd /你的/项目路径
git init
git config user.name "Your Name"
git config user.email "you@example.com"
```

创建 `.gitignore`（忽略编译产物）：

```bash
cat > .gitignore << 'EOF'
build/
*.o
*.out
*.a
*.so
*.exe
compile_commands.json
.vscode/
EOF
```

首次提交：

```bash
git add .
git commit -m "Initial commit"
```

User: 我远程github没有仓库，可以通过git创建么？

GitHub Copilot: 可以，**但要分两种情况**：

## 可以通过命令行创建 GitHub 仓库（推荐）

纯 `git` 本身只能管理本地仓库，  
要“创建 GitHub 远程仓库”通常用 **GitHub CLI (`gh`)**。

先安装并登录：

```bash
sudo apt update
sudo apt install gh -y
gh auth login
```

在你的项目目录执行（会自动在 GitHub 创建远程仓库并推送）：

```bash
cd /你的项目目录
git init
git add .
git commit -m "Initial commit"
gh repo create 仓库名 --public --source=. --remote=origin --push
```

私有仓库把 `--public` 改成 `--private`。

---

## 如果不用 `gh`，也可以先在网页创建再用 git 绑定

1. 去 GitHub 网页点 **New repository** 创建空仓库  
2. 本地执行：

```bash
git remote add origin git@github.com:你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

---

一句话总结：  
- **本地仓库：`git` 就够了**  
- **远程 GitHub 仓库：用 `gh`（命令行）或 GitHub 网页创建**

## 结论

- ✅ **GitHub 仓库已经创建成功了**  
  日志里有：`Created repository gengxun-henu/python_learning`，并且给了仓库链接。
- ❌ 失败的是**后续推送**，不是创建仓库失败。

## 这个错误是什么意思

报错：

`Failed to connect to github.com port 443 ... 连接超时`

意思是：你的本机（或当前网络环境）访问 GitHub 的 HTTPS（443 端口）超时了。  
常见原因是网络不通、代理没配置好、被防火墙限制等。

## 你可以这样排查

先测网络连通性：

```bash
curl -I https://github.com
```

如果这里也超时，就不是 Git 命令问题，而是网络问题。

再检查远程地址：

```bash
git remote -v
```

## 可选解决方案

### 方案 A：修复 HTTPS 网络（最直接）

- 检查代理/VPN是否可用
- 如果配置过错误代理，先清掉：

```bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 方案 B：改用 SSH 推送（很多时候更稳）

先生成并配置 SSH key（如果还没配）：

```bash
ssh-keygen -t ed25519 -C "you@example.com"
cat ~/.ssh/id_ed25519.pub
```

把公钥加到 GitHub 后，改远程地址：

```bash
git remote set-url origin git@github.com:gengxun-henu/python_learning.git
git push -u origin master
```

> 你的本地分支是 `master`，仓库默认分支是 `main`，这不影响首次推送；后续你也可以统一成 `main`。

## SSH 版（推荐，通常比 HTTPS 稳）

### 1) 生成密钥并加入 GitHub

```bash
ssh-keygen -t ed25519 -C "你的GitHub邮箱"
cat ~/.ssh/id_ed25519.pub
```

把输出的公钥粘贴到 GitHub：`Settings -> SSH and GPG keys -> New SSH key`

### 2) 测试 SSH

```bash
ssh -T git@github.com
```

### 3) 在项目目录初始化并推送（统一用 `main`）

```bash
cd /你的项目目录
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin git@github.com:gengxun-henu/python_learning.git
git push -u origin main
```
